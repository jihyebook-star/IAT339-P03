Le Cakery

Project 02: Le Cakery, was a company website development project completed in IAT 235, focused on designing and building a responsive site that reflects a real-world brand identity.

Within my team, I took primary responsibility for the webpage design process, creating Figma prototypes and translating them into a functional website using HTML and CSS. Throughout the project, I explored multiple design directions using the Atomic Design methodology, analyzing and incorporating interface patterns from existing company websites to ensure usability and visual consistency. I also gathered feedback from multiple user groups, including peers, unfamiliar users, and the instructor, and continuously integrated this input into iterative design improvements. GitHub and GitKraken were used to manage version control and support collaborative workflow.

The project aimed to develop a functional and efficient website for an imagined company by applying core web development and design skills. It progressed through key stages: initial ideation and planning, component design and iteration, development in Visual Studio Code, feedback collection, and final refinement. Each phase built on the previous one, allowing for continuous improvement and alignment with user needs.

A key challenge I encountered was limited participation from my teammate during the design phase, which required me to independently manage both design and implementation. To address this, I took initiative in organizing the workflow, actively sought external feedback, and maintained steady progress through structured iterations.

Through this experience, I strengthened my ability to integrate feedback effectively, improved my understanding of user-centered design, and developed adaptability in low-collaboration situations. When presenting this project, I would tailor my explanation to the audience, emphasizing user experience, technical implementation, or design strategy, while clearly outlining each stage with supporting visuals.
